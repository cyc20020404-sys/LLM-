知识库使用说明（RAG）
====================

本目录用于存放 RAG 检索所依赖的知识库文件。支持多种常见格式：

- PDF：`*.pdf`
- Word：`*.docx` / `*.doc`
- Markdown：`*.md` / `*.markdown`
- 文本：`*.txt` / `*.text`
- 其他常见办公文档：通过 Unstructured 兜底解析

使用方式
--------

1. 将你希望被检索的资料文件直接放入本目录（或子目录）下，例如：

   - 心理健康相关科普
   - 情绪管理方法
   - 产品说明文档
   - FAQ、使用手册等

2. 启动 `streamlit_app.py`，在左侧边栏勾选 **「启用 RAG 知识检索」**。

3. 机器人在回答时会优先从这些文件中检索相关段落，把它们作为**参考资料**注入 Prompt，
   再用「小团团」的语气整合成最终回答。

注意事项
--------

- 首次添加或修改文件后，RAG 会自动重建向量索引，时间取决于文件体积。
- 若没有任何文件或解析失败，RAG 会自动降级为普通对话模式，不会报错。
- 为了获得更好的检索效果，建议：
  - 文档内容尽量是**可复制的文字**（扫描 PDF 可能需要 OCR 才能识别）；
  - 控制单个文件不要过于巨大，可按章节拆分成多个文件。
- **老版 Word（.doc）** 会通过 Unstructured 解析；若某 .doc 文件未被加入 RAG，可将其用 Word 另存为 **.docx** 再放入本目录。

# 知识库文件夹

将需要检索的文件放入此目录，启动 Streamlit 应用时会自动加载并建立索引。

## 支持的文件格式

| 格式 | 后缀 |
|------|------|
| PDF | `.pdf` |
| Word | `.docx`, `.doc` |
| Markdown | `.md`, `.markdown` |
| 纯文本 | `.txt` |
| 其他 | 尝试自动解析 |

## 使用方法

1. 将文件放入本目录（支持子目录）
2. 启动 Streamlit 应用
3. 在侧边栏开启「RAG 知识检索」
4. 正常对话即可，模型会自动检索相关内容辅助回答

## 注意

- 首次加载或文件变化时会自动重建索引（需要几秒到几分钟，取决于文件数量）
- 索引缓存在 `rag_index/` 目录，无需手动管理
- 本文件（README.md）不会被加载到知识库中
- **`.doc`（老版 Word）** 依赖 Unstructured 解析；若加载失败，可改为将文件另存为 `.docx` 再放入知识库
